# Contributing Guidelines

Thank you for your interest in contributing to this project.

## Reporting Issues

Please use GitHub Issues for:

- Bug reports  
- Feature requests  
- Improvement suggestions  
- General questions  

When creating an issue, try to include:

- What happened  
- What you expected to happen  
- Steps to reproduce the issue  
- Relevant logs, screenshots, or version information  

Clear and detailed reports help a lot.

## Creating Pull Requests

Pull requests are welcome.

Please make sure to:

- Clearly describe **what** was changed and **why**  
- Keep changes focused and relevant  
- Test your changes if possible  
- Reference related issues if applicable  

## Style Guidelines

- Keep the code clean and readable  
- Avoid unnecessary formatting changes  
- Use comments only where they add real value  
- Prefer simple solutions over complex ones  

## Code of Conduct

This is a community project.  
Respectful and constructive behavior is expected at all times.

Thank you for contributing.
